Bienvenido a mi repositorio de portfolio personal. He desarrollado este proyecto para reunir en un mismo lugar información sobre mí, mi formación y los proyectos en los que he trabajado.

## 🏗️ Filosofía del proyecto

Este portfolio nace como parte de un proyecto del máster en ciberseguridad. Para desarrollarlo he utilizado únicamente HTML y CSS, sin frameworks ni plantillas externas. De esta forma refuerzo conocimientos básicos.

Como vengo de un entorno más relacionado con los sistemas electromecánicos y el sector del motor, siempre me ha parecido importante entender primero los fundamentos antes de apoyarse en herramientas más avanzadas. He intentado aplicar esa misma idea al desarrollo de este sitio web.

El resultado es una web sencilla, ligera y fácil de mantener, centrada en mostrar mi perfil y mis proyectos de forma clara y directa.

## ⚙️ Stack y arquitectura técnica

Para este proyecto he trabajado con tecnologías web fundamentales, priorizando la simplicidad, la organización del código y el rendimiento.

* **HTML5:** La estructura de la página está construida utilizando etiquetas como `main`, `section`, `article` y `nav`, lo que facilita la organización del contenido y mejora la accesibilidad.

* **CSS3 y variables globales:** Los estilos se gestionan mediante variables definidas en `:root`, lo que permite mantener una apariencia coherente y simplifica futuras modificaciones del diseño.

* **Flexbox y CSS Grid:** La distribución de los elementos se adapta a diferentes tamaños de pantalla utilizando las herramientas nativas de CSS para crear diseños flexibles y responsivos.

* **Interactividad sin JavaScript:** Algunos elementos interactivos, como el menú de navegación para dispositivos móviles, he implementado únicamente con HTML y CSS. De esta forma se reduce la complejidad del proyecto y se demuestra que muchas funcionalidades pueden resolverse sin depender de bibliotecas o código adicional.

Más allá de las tecnologías utilizadas, el objetivo principal ha sido mantener una estructura clara, un código fácil de entender y una experiencia de usuario fluida en cualquier dispositivo.

## 🚀 Optimizaciones y buenas prácticas

Además del desarrollo de la interfaz, he intentado aplicar algunas buenas prácticas relacionadas con el rendimiento, la accesibilidad y la visibilidad de la página en internet.

* **SEO y compartición en redes sociales:** He configurado etiquetas Open Graph y Twitter Cards para que el portfolio muestre una vista previa adecuada al compartirlo en plataformas como LinkedIn, X o WhatsApp.

* **Optimización de la carga:** Los elementos más importantes de la página se cargan con prioridad para mejorar la experiencia inicial del usuario, mientras que las imágenes situadas fuera de la primera pantalla utilizan carga diferida (`lazy loading`) para reducir el tiempo de carga.

* **Accesibilidad:** He añadido atributos `aria-label` en los elementos necesarios, revisado el contraste de colores y mantenido una estructura semántica que facilite la navegación mediante lectores de pantalla y otras tecnologías de asistencia.

Aunque se trata de un proyecto académico, he procurado aplicar prácticas habituales del desarrollo web moderno para crear una página más rápida, accesible y preparada para diferentes entornos de uso.

## 💻 Instalación local

Al tratarse de una página web estática desarrollada únicamente con HTML y CSS, no es necesario instalar dependencias ni realizar procesos de compilación.

1. Clona el repositorio:

   ```bash
   git clone https://github.com/CaarlosMrtnz/portfolio-dev.git
   ```

2. Accede al directorio del proyecto:

   ```bash
   cd portfolio-dev
   ```

3. Abre el archivo `index.html` en tu navegador o utiliza un servidor local si lo prefieres.

Con estos pasos podrás visualizar el portfolio y revisar todo el código fuente de forma local.
